JSON Formatting Guidelines¶

The following general guidelines for preparing JSON documents is not intended to replace a proper JSON specification. Documentation of JSON specifications is available at http://json.org/, but this guide and a few examples of files which meet JSON specifications may be adequate for instruction in developing JSON data exchange files.

Basic JSON Formatting Rules¶

  1. JSON ignores indentation and white-space during interpretation. WARES exports include programming style indentation for clarity.
  2. Each complete JSON transmission is enclosed in brackets, and repeated structures within a JSON message are enclosed in brackets ([]).
  3. Each simple data element is named by a preceding identifier label enclosed in quotes (”) and terminated with a colon (:).
  4. Successive data elements are separated by commas (,).
  5. Where multiple data elements are grouped into a data structure, the structure grouping is enclosed in braces ({}).
  6. Where a data structure may be repeated, the set of repeated structures is given a label and the set is enclosed in brackets ([]).
  7. Each element is enclosed in quotes (”). Use an escape sequence (\”) to include a quote within the element text.
  8. null values: JSON does not permit labels associated with blank elements. Either omit the label and element, or provide a null value.
  9. Control characters such as new lines (\n) may be included in the data stream as escape sequences. Each escape code must be preceded by an additional escape indicator, the “\”. See the Escape Sequences table below.

Escape Sequences¶

Following is a reference list of standard escape sequences used to embed special characters or control codes within text strings. While these sequences are valid within JSON data strings, only a few will be meaningful in the context of data exchange.
